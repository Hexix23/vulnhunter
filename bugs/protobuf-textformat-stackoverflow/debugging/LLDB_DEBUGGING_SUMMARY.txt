VULNERABILITY ROOT CAUSE
================================================================================

Vulnerable Code Path: google::protobuf::CodedInputStream::ConsumeFieldMessage()

Issue:
  • recursion_limit_ = 2147483647 (INT_MAX)
  • No actual depth validation during parsing
  • Each recursive call allocates stack frame (~175 bytes on ARM64)
  • Stack exhaustion inevitable with deep nesting
  • No exception recovery possible when stack exhausted

The Bug in Pseudocode:
  class CodedInputStream {
      int recursion_limit_;     // = INT_MAX
      int recursion_depth_;     // Unchecked

          recursion_depth_++;

          parseNestedMessage();
          recursion_depth_--;
      }
  };
