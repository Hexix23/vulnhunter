## Phase 1: Discovery

Perform comprehensive reconnaissance of the target. Both static analysis and active probing.

### Static Analysis
1. Map all code entry points, routes, and endpoints
2. Identify authentication/authorization mechanisms
3. Find hardcoded secrets, API keys, credentials
4. Analyze data flows and trust boundaries
5. Identify dangerous functions and patterns

### Dynamic Analysis (if applicable)
1. Enumerate endpoints and parameters
2. Identify technology stack (headers, responses)
3. Map API structure and authentication flows
4. Check for exposed debug endpoints
5. Test for information disclosure

### Tools You Can Use
- curl, wget for HTTP requests
- nmap for port scanning (if URL target)
- ffuf, dirb for directory enumeration
- nuclei for known vulnerability checks
- Custom scripts as needed

### Output

Identify the **top 10 most promising attack vectors**.

For each vector, document:
```
[FINDING] severity: title
Vector: [name]
Type: [XSS|SSRF|IDOR|SQLi|RCE|Auth Bypass|etc.]
Location: [file:line or endpoint]
Reason: [why this is exploitable]
Dynamic Test: [if you tested it, what was the result]
```

Prioritize by:
1. Impact (RCE > Auth Bypass > Data Leak > XSS)
2. Exploitability (no auth > auth required)
3. Google VRP scope alignment

### Rules
- Be thorough - check everything
- If you can test it dynamically, do it
- Document as you go - use [FINDING] prefix for discoveries
- No time limit - take as long as needed
